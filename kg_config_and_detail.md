# AI+法律论文知识图谱抽取与分块优化（基于给定样本文本）

## 1. 三元组（模板）如何定义

面向“人工智能 + 法律/伦理 + 学术出版”论文，建议采用**核心三元组 + 证据属性**的设计。

### 1.1 实体类型（建议）

- 技术/对象：`AIGC`、`人工智能`、`智能出版`
- 场景/领域：`学术出版`、`同行评议`、`编辑加工`、`成果发布与传播`
- 风险：`版权归属争议`、`虚假信息生成`、`学术生态风险`
- 治理措施：`原则约束`、`义务规范`、`技术检测`、`法律规制`、`教育监督`
- 主体：`学者`、`出版机构`、`国家`
- 政策/文件：`《中共中央关于制定国民经济和社会发展第十五个五年规划的建议》`
- 目标/价值：`营造良好文化生态`、`提升文化原创能力`、`推动高质量发展`

### 1.2 关系类型（建议）

- `应用于`：技术 -> 场景
- `引发/带来`：技术或实践 -> 风险
- `聚焦于`：研究/学者 -> 议题
- `属于`：对象 -> 分类
- `包括`：集合 -> 子项
- `提出/强调`：政策文件 -> 目标/要求
- `影响`：因素 -> 主体/生态
- `需要/依赖`：目标 -> 治理措施
- `助力`：措施 -> 目标

### 1.3 核心模板（可直接用于抽取）

1. 技术应用类  
`(技术, 应用于, 场景)`

2. 风险触发类  
`(技术/实践, 引发, 风险)`

3. 研究关注类  
`(研究主体, 聚焦于, 议题)`

4. 治理构成类  
`(风险治理, 包括, 治理措施)`

5. 政策要求类  
`(政策文件, 提出, 要求/目标)`

6. 影响路径类  
`(AIGC应用, 影响, 学术出版主体/内容/生态)`

7. 目标支撑类  
`(伦理治理路径, 助力, 学术出版高质量发展)`

### 1.4 推荐“增强四元信息”（便于法律场景溯源）

在三元组外附加属性字段（不改变图谱结构）：

- `source_doc`：文档ID
- `source_span`：原文证据片段
- `section`：章节（如“二、伦理风险内涵及分类”）
- `confidence`：抽取置信度
- `time/policy_version`：时间或政策版本

示例（JSON风格）：

```json
{
	"triple": ["AIGC", "引发", "版权归属争议"],
	"source_doc": "paper_xxx",
	"source_span": "部分学者重点关注AIGC辅助学术创作中的版权归属...",
	"section": "研究现状",
	"confidence": 0.91
}
```

## 2. 基于给定内容的分块优化建议

你给出的 [3] 与 [4] 片段存在典型的 PDF/OCR 噪声与跨页重复，建议在切块前做“轻清洗 + 去重 + 结构切分”。

### 2.1 先做文本规范化（强烈建议）

1. 清理异常空格（字内空格）
- 例如：`运 用于` -> `运用于`、`单 一` -> `单一`、`建 议` -> `建议`

2. 清理版面噪声
- 删除页眉页脚类噪声：如 `出版观察 2026.3 ＿ 71`
- 删除孤立页码、横线、装饰符（如全角下划线）

3. 统一标点与引号
- 统一中文引号、括号，减少因异形符号导致的切块抖动

4. 处理脚注标记
- `④ ⑤ ⑥` 这类可映射为引用占位符（如 `[ref_4]`），避免污染语义句子

### 2.2 去重与重叠控制（针对 [3]/[4]）

1. 段落级近重复去重
- [3] 与 [4] 含大量重叠段落，建议用 MinHash 或 SimHash 做近重复检测
- 阈值建议：段落相似度 `>= 0.85` 视为重复，仅保留信息更完整的一段

2. 切块重叠比例优化
- 当前场景建议：`chunk_size=500~800` 中文字符，`overlap=60~120`
- 政策/定义密集段落可提高 overlap；叙述段落可降低 overlap

### 2.3 结构化切分（优先级高于固定窗口）

1. 先按标题切分
- 如：`一、`、`二、`、`三、`，以及“伦理风险内涵及分类”等标题

2. 再按句群切分
- 保持“定义句 + 解释句 + 例证句”尽量在同一块，提升抽取准确率

3. 最后做长度回退
- 若块太长再按标点回退切分，而不是硬截断

### 2.4 面向知识图谱抽取的切块补充

1. 每块保留元数据
- `paper_title`、`section_title`、`paragraph_id`、`policy_doc_flag`

2. 增加“关系触发词”标注（可选）
- 如：`提出`、`包括`、`导致`、`影响`、`助力`
- 便于后续用规则 + 模型混合抽取三元组

## 3. 可直接落地的配置建议（首版）

```yaml
preprocess:
	remove_header_footer: true
	normalize_intra_word_spaces: true
	normalize_punctuation: true
	footnote_marker_mode: replace   # remove | replace
	near_duplicate:
		enabled: true
		method: minhash
		threshold: 0.85

chunking:
	strategy: heading_then_sentence
	chunk_size: 700
	chunk_overlap: 100
	min_chunk_size: 180
	fallback_split: punctuation

kg_extraction:
	entity_types:
		- 技术
		- 场景
		- 风险
		- 治理措施
		- 主体
		- 政策文件
		- 目标
	relation_types:
		- 应用于
		- 引发
		- 聚焦于
		- 包括
		- 提出
		- 影响
		- 助力
	attach_evidence: true
	confidence_threshold: 0.75
```

以上配置适合作为你当前项目的第一版基线，后续可按检索召回率、三元组准确率再迭代阈值。
